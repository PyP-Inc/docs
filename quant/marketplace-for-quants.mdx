# Marketplace for Quants

PyP marketplace support matters for quant because research value is not only about running your own account. It is also about productizing an edge.

## The basic flow

1. train a quant job
2. inspect the artifact
3. create a quant release
4. run simulation and PPE / PPE-PP
5. publish the release
6. create a marketplace listing

## Why the marketplace should be release-based

Quant listings should not be created from raw code or raw jobs.

They should be created from:

- a versioned release
- a completed simulation run
- listing metadata and pricing

That is what lets a buyer see:

- what was validated
- what version is being sold
- what runtime and deployment shape they are subscribing to

## What subscribers should receive

Subscribers should receive signals, not your source code.

That means PyP's value is not only in training. It is in:

- managed delivery
- runtime control
- release integrity
- subscriber management
- billing

## Best practice

- publish only validated releases
- write release notes clearly
- keep listing claims tied to PPE evidence
- price the strategy like a product, not like a vague promise

Quant credibility in a marketplace comes from proof and operational clarity.
